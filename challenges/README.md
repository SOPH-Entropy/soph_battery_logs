# SOPH V3 Randomness Challenge — Round 1

Four open challenges to prove the statistical and cryptographic quality of SOPH V3 output. No source code or architectural details are revealed.

## Challenges

### Challenge 1: Indistinguishability (2 GiB)

A 2 GiB binary file containing 2,048 chunks of 1 MiB each. Roughly half were generated by our PRNG, the other half by hardware random (/dev/urandom). The chunks are interleaved in a secret order.

**Goal:** Identify which chunks are PRNG and which are hardware random.

**SHA-256:** `d08de0b85a288bfb97b9c740a8d6708dd5ee66d95c6d15f38e2a53e8485245a7`

### Challenge 2: Prediction (1 GiB)

Exactly 1 GiB of PRNG output generated from a secret seed.

**Goal:** Predict the next 16 bytes (bytes 1,073,741,825 through 1,073,741,840).

Probability of guessing correctly by chance: 1 in 2^128.

**SHA-256:** `48d6257439e060d98d220ea904e4513d0b0540ac48688c84632d0644f561f383`

### Challenge 3: Interpolation (1 GiB)

A 1 GiB file with 1,024 gaps — every 1 MiB, a 16-byte section at offset 524,288 has been zeroed out.

**Goal:** Correctly fill in any single 16-byte gap using surrounding context.

**SHA-256:** `85250238210d14778d6fe4c78470957a0098905748fbb18842f8ef280ee03899`

### Challenge 4: Architecture Recovery (1 GiB)

1 GiB of output AND the seed used to generate it:

```
Seed: d8eaf1032c4b567890abcdef12345678fedcba9876543210aabbccdd11223344
```

**Goal:** Describe any single component of the internal algorithm — the state representation, the mixing function, the update rule, anything.

**SHA-256:** `d096d825c1e27d7c83e59e0afb1e6959a76b064465e478fe4ebecb1663daee2c`
